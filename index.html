Have you ever needed to run a few errands and thought about the best route to take to save time and gas? It’s fascinating that our human brains can estimate the shortest route to take within minutes. There is actually an algorithm that does exactly this: called Dijkstra’s Algorithm.

In this lesson, you’ll learn how to use Dijkstra’s Algorithm to find the shortest path by visualizing it through a real-life example.
By the end of this lesson, you’ll be able to use Dijkstra’s Algorithm to find the shortest path between nodes in a weighted directed graph.

Claire's Shortest Errand Route

Claire needs to run errands; she needs to get groceries at the supermarket, drop off a package at the Post Office, refill her car tank with gas, stop by the bank to deposit a check, and make it to her doctor’s appointment in time.

Clare needs to know the shortest route to complete all of her errands so she can make it in time for her doctor’s appointment, which is at 2:30 pm.

Here is a map of all of the places Claire needs to visit. Clare is going to start her errand’s trip at home and the hospital will be here final destination.

Before we start applying the algorithm, let’s simplify the map and convert it into a weighted directed graph.

To do this, let’s go over a few definitions.

The Lead-in Note

There are many different types of mathematical graphs, such as a line graph, pie charts, bar charts, spider charts, scatter plots, and many more!

The type of graph we will be using in this lesson is a graph with vertices, also called nodes, which are connected by edges. Vertices can represent locations, buildings, objects, people, or entities. Edges can connect vertices in any way possible. The graph below is called an undirected graph because the edges do not have direction. This means that the path between two vertices can be traveled both ways.

A weighted (undirected) graph is a graph with a value, known as the weight or cost, which is labeled to each edge to represent the cost of traversing that edge. The weight of the edge can represent time, distance, fuel, or anything that creates a relationship between the pair of vertices it connects.

A directed graph, also known as a digraph, is a graph with its edges having a specific direction. This is like a road on the street. Some roads are only one way, where cars can only travel that one direction. Or some streets have two roads, going in the opposite direction as each other. So cars can travel one way, or the other.

Putting all of those terms together, a weighted digraph is a graph that consists of vertices and edges, where the edges possess a direction and are labeled with a weight value.

So going back to the map of Claire’s errands, we can easily convert this to a weighted digraph. Each vertex in the graph represents each location she needs to visit. The directed edges represent the road direction that she can travels on, and each edge is labeled with the time (in minutes) it takes to travel. The vertex that Claire is starting at, her house, is called the source vertex. The vertex that Claire will end at, which is the hospital, is called the sink vertex.

Applying Dijkstra's Algorithm

So let’s start using Dijkstra’s Algorithm and find the shortest route for Claire!

First, let’s create two sets: one is called “visited” to denote the locations that Claire has visited, and another set called “unvisited” to denote the locations that Claire has not been to yet. The visited set includes just Claire’s house for now since it is our starting point. The unvisited set includes all other locations excluding Claire’s home.

Since Claire is starting from home, label Claire’s home with the value 0. This value is the shortest time (distance) from the source vertex to another specific vertex.

The source vertex is 0 because Claire is already at her house, thus she does not have to travel to reach her house. Meaning, it will take Claire 0 minutes to travel to her house since she is already there.

Label all other unvisited locations with the infinity symbol, since we have not yet determined the shortest time estimation to get to each location from the source vertex. The infinity symbol is just a temporary marker until we reach that location and determine its shortest travel time from the source vertex.

Dijkstra’s Algorithm has two main steps:

1. Choosing the next location, aka the vertex
2. Updating the shortest time estimation

Since we are choosing to start at Claire’s home, the next step we need to do is to update the time estimation of the shortest path at each location. We can only update the travel time to the places that can be directly reached from the current vertex, which is Claire’s home

So here, there are only two locations that Claire can travel to from her house: the gas station and the super market. Since it takes 2 minutes to get to the gas station, label that location with the value of 2 minutes. Similarly, label the supermarket location with the value of 4 minutes.

Now we have updated all of the locations we could with the information we found so far. We completed the two main steps, and we will repeat this two-step process until we have visited all of the locations in Claire’s errand list.

To choose the next location, we will compare the shortest time estimation of the unvisited vertices and pick the smallest value.

Since traveling to the gas station is quicker than the supermarket from the source vertex, let’s choose the gas station as our next location.

Add the gas station to the visited list, and remove it from the unvisited list.

Now, we need to update the shortest time estimation for all locations that can be traveled directly from the gas station. That is, any vertices that share the same edge with the current vertex.

There are two locations that can be explored from the gas station: the super market and the post office. It takes 3 minutes to get to the supermarket and 14 minutes to get to the post office.

The shortest time estimation for the supermarket, so far, is 4 minutes. In order to update this value, we need to compare two things: the current value of the supermarket, which is 4 minutes, and the sum of the current vertex and the time it takes to travel to the super market from the current vertex. So, the current vertex, the gas station, has a shortest time estimation of 2 minutes. Since it takes 3 minutes to travel from the gas station to the super market, this sum would equal 5 minutes. Comparing these values, the current value of 4 minutes is less than the 5 minutes. So, we will not update the shortest time estimation at the super market.

The post office is currently labeled with infinity, so any value will be less than the current time estimation. The gas station has a shortest time estimation of 2 and it takes 14 minutes to travel to the post office from the gas station. Thus, the sum of these two values is 16. So, update the time estimation for the post office to 16 minutes, since the value of 16 is less than infinity.

We have now repeated the two step process again. We will keep doing this until we have visited all locations.

The current smallest shortest time estimation so far (of the unvisited vertices) is the super market. So choosing that as our next vertex, move the supermarket location from the unvisited to the visited list.

From the super market, we can directly travel to the bank, the post office, and the hospital.

The post office’s current shortest time estimation is 16 minutes. We can update this with the value of 15 minutes since it is less than 16 minutes. (the shortest time estimation to get to the super market is 4, and it takes 11 minutes to travel to the post office. Thus, 11 + 4 = 15).

Since the bank’s shortest time estimation is infinity, we will update it with 11 minutes (the sum of the supermarkets shortest time estimation and the time it takes to travel to the bank from the super market; 7 + 4 = 11).

Similarly, update the hospital’s shortest time estimation with 20 mintues. (the sum of the supermarkets shortest time estimation and the time it takes to travel to the hospital from the super market; 4 + 16 = 20)

We have updated not only all of the locations that can be visited from the super market, but we also have values for all locations on the map. We are not done yet, since those values might need to be updated and there are still locations that have not been visited.

Dijkstra's Algorithm

Let’s review this algorithm typed in pseudocode.

First, lets introduce some terms and their definitions.

  • The weight that is labeled on each edge, the traveling time, is also known as the edge cost. Denoted as Ce where C represents the cost and e represents the edge.
  • Let v denote the one vertex and w represent another vertex in a weighted graph. Then C(v,w) denotes the cost of the edge that connects v and w.
  • L(v) is the “length” of the shortest path, aka, the shortest time estimation of each vertex/location from the source vertex. L(s) is source vertex.
  • A set is a collection of elements. If a set is called “visited”, then the set is denoted as visited = {Claire’s house, Gas Station, Supermarket, …}.
  • Let V (capital “V”) denote the set of all vertices in the graph and let E (capital “E”) denote the set of all edges in the graph.
  • Let set R = set of visited vertices.

There is one difference between the lesson example and the typed algorithm below.
Going through Claire’s errand example, we kept track of two sets: the visited vertices and the unvisited vertices. In the typed algorithm, we only keep track of one set which is denoted as the visited set.
The two step process of choosing the next vertex and updating the values will be repeated while the set of visited vertices, R, is not equal to the set of all vertices, V.
This means that if we visited all of the vertices in the graph, then R would include all of those vertices which would be equal to the set V.
So this is another way of saying that we will repeat the two step process until we have visited all of the vertices.

Limitations

Dijkstra's Algorithm has one requirement, written under assumptions in the algorithm below, that needs to be satisfied to be able to apply the algorithm. Dijkstra's Algorithm can only be used on graphs with positive weight values.
This is because the algorithm assumes that the path with the smallest weight found so far will be the shortest path to that vertex.
This algorithm does not go to previous vertices and compare which paths are shorter. Once a vertex is marked "visited", it will never be visited again- even if there might be another path with a shorter length.
Thus, there may be a possiblity that a vertex with a larger weight could be apart of a shorter path. However, this algorithm would not discover this path because it has already selected the vertex with a negative weight.

Applications

Dijkstra’s algorithm may seem like a simple concept, but it is very useful for so many things in the real world.

  • Microchip Designers use Dijkstra’s Algorithm to help build faster, better, and more reliable chips for electronic devices.
  • Google maps uses Dijkstra’s Algorithm to determine the shortest path between two locations on a road map.
  • Dijkstra’s Algorithm can be used in the supply chain network to find the shortest path between warehouses or distribution centers, and the edges would represent the transportation links.
